The process

  • The application History singleton fires a ValueChangeEvent, from a url change (or onload)
  • The RegistryHistoryMapper singleton determines via pattern matchinwhich BasePlaceTokenizer can parse the url, and fires a PlaceChangedEvent containing the parsed BasePlace
  • The ActivityMapper singleton determines which Activity (a Dirndl model) to display, that activity is rendered and either replaces the DOM body contents (if it's aModalActivity), or replaces a content element somewhere in the DOM (such as in a header/content/footer app layout).
