Dirndl names

Goals

  • Minimise rendered html size
  • Mininmise generated css size
  • Make html legible (tags reflect model classes)

Guidelines

  • If the rendered model class has no subclasses, don't use a tag or cssClass attribute (rendered tag name will be the css-ified simple class name)
  • If the rendered model class has subclasses, but no UI attributes (css styles) which should be shared by subclasses, ditto
  • Otherwise: if it's known that sub-classes have no subclass-specific styles -and- subclass tag names wouldn't appreciably improve html legibility , set the tag attribute of the parent class @Directed annotation
  • Otherwise set the cssClass attribute of the parent class @Directed annotation - almost always don't use scss mixins to replicates styles to the different subclass tags (unless, say, the hierarchy is final/sealed - but even then the resultant css is probably )

Example - the Dirndl Tree

Both the container - Tree - and the node class - TreeNode are intended for subclassing. Both base classes have styles. Since the container occurs only once, definitely uss cssClass (not tag) since it will only bulk the html a few bytes and will improve legibility.

Each tree can contain potentially many treenode instances, so using cssClass - <my-node class="node"> - is a little more wordy tha <node"> - but it's necessary for custom styling of subclass nodes - .my-node label {color: blue};
