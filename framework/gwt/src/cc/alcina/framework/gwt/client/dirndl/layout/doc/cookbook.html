The Dirndl Cookbook

...or how do I wrestle this dang thing into what I want?

The @Directed annotations aim to produce sensible results, via sensible defaults for the most common use cases - but that means the uncommon cases require a little more wrangling:

Common cases

Rendering a java bean (any class with getters)

Transforming the entries of a list

To transform the elements of a list, add the @Directed.Transform( <transform-class>) annotation - e.g. to transform List<String> to List<HtmlModel>, add @Directed.Transform(LeafModel.HtmlModel.Transform.class)

Wrapping a container in more than one tag

Transforming an object

Rendering a java bean without a containing tag

Uncommon cases

Transforming a list

The simplest transform annotation, applied to a list property, transforms the elements, not the containing list. To transform the parent list, you must explicitly set the renderer on the annotation:

		
//transforms each Entry element of the list - not what you want (since the renderer is the
// default for a java.util.Collection subtype, DirectedRenderer.Collection)
@Directed.Transform(Tables.Multiple.class)
public List<Entry> getEntries() {
  return this.entries;
}
